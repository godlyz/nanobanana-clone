## 审核员职责

### 核心职责

作为Nano Banana社区审核员，你的主要职责包括：

1. **内容审核** / Content Moderation
   - 审查举报的作品、评论、博客文章
   - 主动巡查公共内容，发现潜在违规
   - 确保内容符合社区规范

2. **用户管理** / User Management
   - 处理用户举报
   - 发布警告、限制或封禁账号
   - 处理用户申诉

3. **社区建设** / Community Building
   - 维护积极、友好的社区氛围
   - 回答用户疑问
   - 收集用户反馈

4. **文档和报告** / Documentation and Reporting
   - 记录审核决策和原因
   - 每周提交审核报告
   - 识别和上报系统性问题

### 工作时间要求

- **志愿审核员** / Volunteer Moderators: 每周至少5小时 / At least 5 hours per week
- **社区管理员** / Community Managers: 每周至少15小时 / At least 15 hours per week
- **全职管理员** / Full-time Moderators: 每周40小时 / 40 hours per week

### 权限级别

| 级别 | 权限 | 说明 |
| **Level 1 - 见习审核员** / Trainee | 查看举报、标记内容 | 无删除/封禁权限，需要主管审批 |
| **Level 2 - 初级审核员** / Junior Moderator | 删除评论、发布警告 | 可删除明显违规评论，可发布警告 |
| **Level 3 - 高级审核员** / Senior Moderator | 删除作品、7天封禁 | 可删除违规作品，可进行短期封禁 |
| **Level 4 - 社区管理员** / Community Manager | 永久封禁、系统配置 | 完全权限，可永久封禁，可修改规则 |


## 审核原则

### 1. 公平公正

- **一视同仁** / Equal Treatment
  - 对所有用户适用相同标准，无论其影响力、粉丝数量或贡献
  - 不因个人喜好影响审核决策

- **基于事实** / Fact-Based
  - 仅根据客观证据做出决策
  - 疑罪从无 - 无法确定时，倾向于保留内容

### 2. 一致性

- **遵循先例** / Follow Precedent
  - 参考过去类似案例的处理方式
  - 保持审核标准的稳定性

- **团队协作** / Team Collaboration
  - 复杂或有争议的案例，咨询其他审核员
  - 定期团队会议，统一标准

### 3. 透明度

- **说明原因** / Explain Reasons
  - 删除内容或封禁账号时，提供明确理由
  - 引用具体违反的社区规范条款

- **记录决策** / Document Decisions
  - 在审核系统中记录每个决策的理由
  - 便于申诉处理和质量审查

### 4. 尊重

- **专业态度** / Professional Attitude
  - 即使处理违规内容，也保持专业、礼貌
  - 不使用侮辱性语言

- **隐私保护** / Privacy Protection
  - 不公开讨论具体用户的审核案例
  - 仅在内部系统中共享敏感信息

### 5. 比例原则

- **适度处罚** / Appropriate Penalties
  - 处罚应与违规严重程度相匹配
  - 首次轻微违规：警告 > 临时限制 > 封禁


## 违规内容识别

### 违规分类

#### 🔴 严重违规（立即行动）

**Level 5 - 严重非法内容** / Serious Illegal Content
- ❌ 儿童性虐待材料（CSAM）
- ❌ 人口贩卖、暴力犯罪
- ❌ 恐怖主义宣传
- **行动**: 立即删除 + 永久封禁 + 报告执法机构

**Level 4 - 严重有害内容** / Serious Harmful Content
- ❌ 真实暴力威胁、仇恨言论
- ❌ 自残、自杀诱导
- ❌ 极端暴力、血腥内容
- **行动**: 立即删除 + 永久封禁（或30天封禁视情况）

#### 🟠 中等违规（24小时内处理）

**Level 3 - 不当内容** / Inappropriate Content
- ⚠️ 成人内容（色情、裸露）
- ⚠️ 恶意欺凌、骚扰
- ⚠️ 虚假信息、恶意误导
- **行动**: 删除内容 + 警告（首次）或7天封禁（再犯）

**Level 2 - 垃圾信息** / Spam
- ⚠️ 重复发布相同内容
- ⚠️ 未经许可的广告推广
- ⚠️ 恶意刷屏
- **行动**: 删除内容 + 警告 + 限制发帖频率

#### 🟡 轻微违规（72小时内处理）

**Level 1 - 低质量内容** / Low-Quality Content
- 💬 无意义评论（如"顶""沙发"）
- 💬 过度使用标签（tag spam）
- 💬 轻微不礼貌行为
- **行动**: 删除内容 + 提示（不记录违规）

### 边界案例判断

**艺术 vs. 色情** / Art vs. Pornography
- ✅ 允许：艺术性裸体（如古典雕塑风格AI作品），无性暗示
- ❌ 禁止：明确性行为、生殖器特写、性暗示姿势

**批评 vs. 骚扰** / Criticism vs. Harassment
- ✅ 允许："这个作品的构图有些问题，建议调整光影"
- ❌ 禁止："你的作品垃圾得要死，别发了"

**灵感借鉴 vs. 抄袭** / Inspiration vs. Plagiarism
- ✅ 允许：使用相似风格但独特创意，标注灵感来源
- ❌ 禁止：直接复制他人prompt生成相同作品，不标注来源

**讽刺幽默 vs. 仇恨言论** / Satire vs. Hate Speech
- ✅ 允许：善意的讽刺和幽默，不针对特定群体
- ❌ 禁止：基于种族、性别、宗教的歧视性"玩笑"

**难以判断时** / When Uncertain
1. 咨询其他审核员（Level 3+）
2. 查看用户历史记录（是否有违规前科）
3. 考虑上下文（发帖时间、地区、文化背景）
4. 倾向于保留内容，但标记为"需要关注"


## 审核流程

### 标准审核流程

用户举报 / User Report
审核队列 / Moderation Queue
审核员审查 / Moderator Review
    ├─ 明显违规 → 立即处理
    ├─ 边界案例 → 咨询团队
    └─ 无违规 → 标记为"已审核"
执行处罚措施 / Execute Penalty
通知用户 / Notify User
记录决策 / Document Decision

### 具体步骤

#### Step 1: 接收举报

**登录审核后台** / Log into Moderation Dashboard
- 访问 `/admin/moderation/queue`
- 查看待处理举报列表

**举报信息包含** / Report Information Includes
- 举报对象（作品、评论、用户）
- 举报类型（垃圾信息、不当内容、版权侵犯等）
- 举报人提供的详细说明
- 举报时间和优先级

#### Step 2: 评估内容

**查看完整上下文** / View Full Context
- 阅读完整内容（作品描述、评论串、用户资料）
- 检查用户历史记录和违规前科
- 查看相关评论和互动

**判断违规类型和严重程度** / Determine Violation Type and Severity
- 参考本手册"违规内容识别"章节
- 确定违规级别（Level 1-5）

#### Step 3: 做出决策

**单人决策** / Solo Decision (Level 1-2 violations)
- 初级审核员（Level 2+）可独立处理明显违规

**团队决策** / Team Decision (Level 3+ violations)
- 复杂或严重案例，需至少2名审核员同意
- 有争议的案例，升级至社区管理员（Level 4）

#### Step 4: 执行处罚

**处罚选项** / Penalty Options
- **删除内容** / Remove Content: 从公开展示中移除
- **警告** / Warning: 发送系统通知，记录违规
- **限制功能** / Restrict Features: 禁止发帖、评论、点赞等（1-30天）
- **封禁账号** / Ban Account: 临时（7-30天）或永久封禁

**执行操作** / Execute Action
- 在审核后台选择相应处罚措施
- 填写处罚理由（必填，用户可见）
- 设置处罚时长（如适用）

#### Step 5: 通知用户

**自动通知** / Automated Notification
- 系统自动发送通知至用户通知中心
- 通知包含：违规内容、违反条款、处罚措施、申诉链接

**邮件通知** / Email Notification (严重违规 / Serious violations)
- Level 3+违规会同时发送邮件通知

#### Step 6: 记录决策

**审核日志** / Moderation Log
- 系统自动记录：
  - 审核员ID和姓名
  - 处理时间
  - 违规类型和级别
  - 处罚措施
  - 处罚理由

**内部备注** / Internal Notes
- 添加额外背景信息（仅审核团队可见）
- 标记特殊情况或需要跟进的案例


## 处理标准

### 违规处罚矩阵

| 违规级别 | 首次违规 | 第二次 | 第三次 | 严重情节 |
| **Level 1** | 删除+提示 | 删除+警告 | 删除+3天禁言 | - |
| **Level 2** | 删除+警告 | 删除+7天禁言 | 删除+30天封禁 | 永久封禁 |
| **Level 3** | 删除+7天封禁 | 删除+30天封禁 | 永久封禁 | 永久封禁 |
| **Level 4** | 永久封禁 | - | - | 报告执法 |
| **Level 5** | 永久封禁+报告执法 | - | - | - |

### 特殊处理情况

**未成年用户** / Minor Users (< 18岁)
- 更严格的内容审核标准
- 禁止成人内容（即使艺术性质）
- 违规时，通知监护人（如有绑定）

**新用户** / New Users (注册<7天)
- 首次轻微违规，给予教育性警告，不记录违规
- 严重违规仍按标准处理

**高价值用户** / High-Value Users (创作者、Pro订阅用户)
- **不降低标准** - 适用相同规则
- 但处理前多一次人工复核，避免误判

**版权争议** / Copyright Disputes
- 不审核版权归属（非审核员职责）
- 仅处理明显侵权（如盗用他人作品）
- 复杂版权问题，转交法务团队

## 案例分析

### 案例1: 垃圾信息（Spam）

**情况** / Situation
- 用户A在10分钟内发布了20条相同内容的评论："关注我，我也关注你！"

**分析** / Analysis
- 违规类型：Level 2 - 垃圾信息
- 首次违规（用户A无违规前科）

**处理** / Action
- 删除所有20条垃圾评论
- 发布警告，告知违反社区规范第4.3条
- 限制评论功能24小时，防止继续刷屏

**通知用户** / Notify User
你好，
我们发现你在短时间内发布了大量重复评论，这违反了社区规范关于垃圾信息的规定。
我们已删除这些评论，并限制你的评论功能24小时。
请阅读社区规范，避免再次违规。


### 案例2: 边界艺术内容（Borderline Artistic Content）

**情况** / Situation
- 用户B发布了一张AI生成的古典雕塑风格作品，包含艺术性裸体
- 有3名用户举报为"成人内容"

**分析** / Analysis
- 内容为艺术性裸体，无性暗示
- 风格类似米开朗基罗的《大卫》
- 作品标注为"古典雕塑风格实验"

**咨询团队** / Consult Team
- 审核员1: 认为是艺术作品，可保留
- 审核员2: 建议添加"可能包含艺术性裸体"警告标签

**处理** / Action
- 保留作品，不删除
- 添加内容警告标签："艺术性裸体"
- 设置为18+内容，需用户确认年龄才能查看
- 回复举报者，说明艺术性裸体与色情的区别

**通知用户B** / Notify User B
你好，
你的作品被部分用户举报为"成人内容"。
经审核，我们认为这是艺术性作品，符合社区规范。
为了照顾不同用户的观感，我们为你的作品添加了"艺术性裸体"标签和18+年龄限制。
继续创作！


### 案例3: 恶意骚扰（Malicious Harassment）

**情况** / Situation
- 用户C在用户D的多个作品下发布侮辱性评论
- "你的作品垃圾得要死""别发了，丢人""滚出这个平台"
- 持续时间：1周，共15条恶意评论

**分析** / Analysis
- 违规类型：Level 3 - 恶意骚扰
- 用户C有1次违规前科（3个月前，Level 1垃圾评论）
- 明显针对特定用户，持续性骚扰

**处理** / Action
- 删除所有15条恶意评论
- 7天封禁用户C账号
- 通知用户D，告知已处理骚扰者
- 内部备注：用户C列入"需要关注"名单，再犯直接永久封禁

**通知用户C** / Notify User C
你好，
我们发现你在过去一周内对用户[D]发布了多条侮辱性评论，构成恶意骚扰。
这严重违反了社区规范第3.1条（禁止骚扰和欺凌）。
我们已删除这些评论，并对你的账号进行7天封禁。
如果再次违规，你的账号将被永久封禁。


## 申诉处理

### 申诉审查流程

**Step 1: 接收申诉** / Receive Appeal
- 用户通过 `/appeal` 页面提交申诉
- 系统自动创建申诉工单

**Step 2: 初步审查** / Initial Review (1个工作日内 / Within 1 business day)
- 检查申诉是否完整（包含申诉理由、证据）
- 查看原审核决策和证据

**Step 3: 重新评估** / Re-Evaluation (3-7个工作日 / 3-7 business days)
- 分配给原审核员之外的其他审核员
- 重新审查案例，考虑申诉人的说明
- 咨询团队（复杂案例）

**Step 4: 做出决定** / Make Decision
- **维持原判** / Uphold: 原决策正确，申诉被驳回
- **撤销处罚** / Overturn: 原决策错误，恢复内容/账号
- **减轻处罚** / Reduce: 原决策过重，减轻处罚（如30天改为7天）

**Step 5: 通知结果** / Notify Result
- 向申诉人发送详细结果通知
- 如撤销，立即恢复内容/账号功能

### 申诉成功率目标

- **目标**: 20-30%的申诉被部分或全部支持
- 如果申诉成功率 < 10%，可能说明审核标准过严
- 如果申诉成功率 > 50%，可能说明初审质量低


## 自我保护

### 心理健康

审核工作可能涉及大量负面内容，保护自己的心理健康非常重要：

**定期休息** / Regular Breaks
- 每审核1小时，休息10-15分钟
- 避免连续审核超过4小时

**寻求支持** / Seek Support
- 如果看到令你不适的内容，可以：
  - 向主管或其他审核员倾诉
  - 使用公司提供的心理咨询服务
  - 申请休息或调换审核类别

**设定界限** / Set Boundaries
- 工作时间外，不处理紧急申诉（除非值班）
- 不将工作压力带回家

### 举报严重内容

**遇到Level 5违规（CSAM、恐怖主义等）** / Encounter Level 5 violations (CSAM, terrorism, etc.)
1. 不要下载或保存内容
2. 立即在系统中标记为"严重违规"
3. 通知社区管理员和法务团队
4. 填写《严重违规报告表》，包括：
   - 内容URL和用户ID
   - 违规类型和严重程度
   - 发现时间
   - 是否已报告执法机构
